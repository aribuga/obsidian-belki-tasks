# Projects and Inbox

## Inbox

Tasks with no project assigned land in **Inbox**. This is a smart view, not a real project. It shows all tasks that have not been organized yet.

To move a task out of Inbox, assign it a project from the task detail view.

---

## Projects

Projects appear in the sidebar under **Projects**. Each project has a colored dot.

### Creating a project

You can create a project from the task detail view or quick-add composer using `//New project name`. Type the name and confirm with the ✓ button.

### Renaming a project

Open a project view and click the **⋯** menu next to the project title. Select **Rename**.

### Archiving a project

Open the project **⋯** menu and select **Archive**. Archived projects move to the **Archived** section at the bottom of the sidebar. Tasks in archived projects are hidden from active views.

### Project colors

Assign a color to each project in Settings → Project colors. Colors appear as dots in the sidebar and on task cards.

---

## Projects view

The **Projects** view shows all projects together. You can click a project name in the sidebar to see only that project's tasks.

### Group by

Inside a project view, you can group tasks by:

- **None** — flat list (default)
- **Label** — one section per label; unlabelled tasks go under "No label"
- **Priority** — P1 → P2 → P3 → P4 → No priority

To change grouping, open the **Sorting** menu and choose a **Group by** option. The setting is saved and persists between sessions. Sorting still applies within each group.

Empty groups are not shown.
