# belki — Test Vault

This vault exists for manual regression testing of the belki Obsidian plugin.
**Do not store personal or private data here.** Everything in this vault is fake test data.

---

## Quick start

```bash
# From the repo root — build the plugin and copy it into this vault:
npm run test:vault

# To reset sample task data back to a clean state first:
npm run test:vault:reset
```

Then in Obsidian:

1. **File → Open Vault** and select `test-vaults/belki-test`
2. Settings → Community Plugins → enable **belki** if not already enabled
3. Click the belki ribbon icon (check-circle) to open the board
4. Work through `manual-regression-checklist.md`

---

## What each script does

| Script | What it does |
|---|---|
| `npm run test:vault` | Builds plugin + copies `main.js`, `manifest.json`, `styles.css` into the vault |
| `npm run test:vault:reset` | Regenerates sample task data for today's date, then does the above |

### After updating plugin files

You do **not** need to close and reopen Obsidian. Go to:
**Settings → Community Plugins → belki → Disable → Enable**

This hot-reloads the plugin with the new build.

---

## Vault structure

```
belki-test/
  .obsidian/
    community-plugins.json   ← enables belki automatically
    plugins/
      belki/                 ← populated by npm run test:vault (gitignored)
  _belki_files/
    Data/
      YYYY-MM.md             ← generated by reset script (gitignored)
  README.md
  manual-regression-checklist.md
```

---

## Notes

- The plugin stores task data in `_belki_files/Data/`. These files are gitignored because the reset script regenerates them with today's date.
- Plugin binaries (`.obsidian/plugins/belki/`) are also gitignored; always run `npm run test:vault` before opening the vault after a fresh clone.
- This vault is committed to the repo so the structure and checklist are version-controlled, but generated data is not.
